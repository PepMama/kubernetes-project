KUBERNETES : PODS, service, volume
-----------------------------
Dans pod :
apiVersion: v1
kind: Pod
metadata:
  name: example-pod
  labels:
    app: example
spec:
  containers:
  - name: nginx-container
    image: nginx:latest
    ports:
    - containerPort: 80

-----------------------------
Dans service :
apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  selector:
    app: example
  ports:
    - protocol: TCP
      port: 80
      targetPort: 80

-----------------------------
Accéser à NGINX sur navigateur :
Port Forward :  kubectl port-forward service/nginx-service 8080:80 -n default

Solution 2 — avec NodePort

Mdifier le service :

apiVersion: v1
kind: Service
metadata:
  name: nginx-service
  namespace: default
spec:
  type: NodePort
  selector:
    app: example
  ports:
    - protocol: TCP
      port: 80
      targetPort: 80
      nodePort: 30080

Dans le terminal :

kubectl apply -f service.yaml

Solution 3 :
apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  type: LoadBalancer
  selector:
    app: example
  ports:

    port: 80
